01 · VERTEX AI SETUP

Vertex AI & Claude 모델 이해

Google Cloud의 Vertex AI 플랫폼에서 Claude를 사용하는 방법과 GCP 인증 설정을 마스터합니다.

이 섹션에서 배울 내용

  • Vertex AI 플랫폼에서 Claude 모델을 활성화하는 방법
  • gcloud CLI를 이용한 GCP 인증 설정 흐름
  • AnthropicVertex 클라이언트 생성과 모델 ID 표기법
  • Vertex AI와 직접 Anthropic API의 차이점 비교
  • API 요청 생명주기: 토큰화 → 임베딩 → 문맥 반영 → 생성

Vertex AI에서 Claude 활성화하기

Claude를 Vertex AI에서 사용하려면 먼저 Google Cloud Console에서 모델을 활성화해야 합니다.

1

Vertex AI Dashboard 접속

console.cloud.google.com/vertex-ai/dashboard에 접속 후 왼쪽 네비에서 Model Garden을 클릭합니다.

2

Anthropic 모델 검색 & 활성화

'Search models' 박스에 Anthropic을 입력하고 원하는 모델을 선택 후 Enable 버튼을 클릭합니다.

3

gcloud CLI 설치 & 인증

아래 명령으로 gcloud CLI를 설치하고 프로젝트를 설정합니다.

4

SDK 설치

pip install "anthropic[vertex]"로 Vertex AI 지원이 포함된 Anthropic SDK를 설치합니다.

# gcloud CLI 인증 및 프로젝트 설정
$ gcloud init
$ gcloud auth login
$ gcloud config set project YOUR_PROJECT_ID
$ gcloud auth application-default login

# Anthropic SDK with Vertex AI 지원 설치
$ pip install "anthropic[vertex]"
💡

Application Default Credentials (ADC): gcloud auth application-default login을 실행하면 로컬 파일에 인증 정보가 저장됩니다. Anthropic SDK는 이 인증 정보을 자동으로 탐지하므로 API 키를 별도로 관리할 필요가 없습니다.

AnthropicVertex 클라이언트 생성

인증 설정이 완료되면 AnthropicVertex 클라이언트를 생성합니다. 직접 Anthropic API와 달리 region과 project_id가 필요합니다.

from anthropic import AnthropicVertex

# Vertex AI 클라이언트 생성 (API 키 불필요)
client = AnthropicVertex(
    region="global",            # 또는 "us-central1" 등 특정 리전
    project_id="your-project-id"  # GCP 프로젝트 ID
)

# 모델 ID — Vertex AI에서는 '@' 기호로 버전을 구분
model = "claude-sonnet-4@20250514"  # ← Anthropic API와 다른 표기!
⚠️

모델 ID 표기법 차이: 직접 Anthropic API는 claude-sonnet-4-20250514 (하이픈), Vertex AI는 claude-sonnet-4@20250514 (@)를 사용합니다. 혼동하지 않도록 주의하세요.

Vertex AI vs. Anthropic API 직접 비교

🔑

중요한 사실: 클라이언트 생성 방식과 모델 ID를 제외하면 client.messages.create() 이후의 모든 API는 완전히 동일합니다. Anthropic API를 알고 있다면 Vertex AI로 전환하는 데 추가 학습이 거의 필요하지 않습니다.

API 요청 생명주기 (Request Lifecycle)

사용자 입력이 Claude 응답으로 변환되는 5단계 과정을 이해하면 더 나은 애플리케이션을 설계할 수 있습니다.

1

Request to Server

클라이언트 앱이 여러분의 서버로 요청 전송 — 절대 클라이언트에서 직접 Vertex를 호출하지 마세요 (보안 위험)

2

Request to Vertex

여러분의 서버가 GCP 인증 정보으로 Vertex AI에 안전하게 요청 전송

3

Model Processing

Claude가 토큰화 → 임베딩 → 문맥 반영 → 생성 4단계로 응답 생성

4

Response to Server

Vertex가 생성 텍스트 + 사용 토큰 + stop_reason 반환

5

Response to Client

여러분의 서버가 클라이언트에게 최종 응답 전달

생각해보기

Vertex AI의 주요 장점은 GCP 생태계와의 통합입니다. Cloud Logging으로 모든 API 호출을 추적하고, VPC Service Controls로 데이터 유출을 방지하고, IAM으로 세밀한 권한 관리가 가능합니다. 여러분의 조직이 이미 GCP를 사용한다면 Vertex AI가 자연스러운 선택입니다.

02 · API REQUESTS

첫 API 요청 & 대화 관리

AnthropicVertex로 첫 요청을 보내고, 멀티턴 대화와 스트리밍을 구현합니다.

이 섹션에서 배울 내용

  • messages.create()로 기본 요청 보내기 및 응답 추출
  • 멀티턴 대화 상태 관리 — 메시지 기록 수동 관리
  • 스트리밍으로 실시간 응답 출력하기
  • 구조화된 출력 (JSON 형식 강제)

핵심 요약

기본 요청
client.messages.create(model, max_tokens, messages) — Anthropic API와 동일한 인터페이스
응답 추출
message.content[0].text — 항상 이 패턴으로 텍스트 추출
상태 관리
Claude는 상태가 없음 — 개발자가 메시지 기록을 직접 유지해야 함

첫 번째 API 요청

from anthropic import AnthropicVertex

client = AnthropicVertex(region="global", project_id="my-project")
model = "claude-sonnet-4@20250514"

# 기본 요청
message = client.messages.create(
    model=model,
    max_tokens=1000,
    messages=[
        {"role": "user", "content": "양자 컴퓨팅이란 무엇인가요? 한 문장으로 설명해주세요."}
    ]
)

# 응답 텍스트 추출
print(message.content[0].text)

# 응답 메타데이터 확인
print(message.usage.input_tokens, message.usage.output_tokens)
print(message.stop_reason)  # "end_turn" 또는 "max_tokens"

응답 구조 이해하기

message.content는 리스트 형태로 반환됩니다. 일반 텍스트 응답은 항상 인덱스 0에, 타입은 "text"입니다. Tool Use 시에는 여러 content 블록이 포함될 수 있습니다.

멀티턴 대화 관리

Claude는 완전히 무상태(stateless)입니다. 이전 대화를 기억하지 못하므로 매 요청마다 전체 대화 기록을 함께 전송해야 합니다.

def add_user_message(messages, text):
    messages.append({"role": "user", "content": text})

def add_assistant_message(messages, text):
    messages.append({"role": "assistant", "content": text})

def chat(messages, system=None, temperature=1.0):
    params = {
        "model": model,
        "max_tokens": 1000,
        "messages": messages,
        "temperature": temperature,
    }
    if system:
        params["system"] = system
    message = client.messages.create(**params)
    return message.content[0].text

# 멀티턴 대화 예시
messages = []
add_user_message(messages, "양자 컴퓨팅이란?")
answer = chat(messages)
add_assistant_message(messages, answer)   # ← 응답을 기록에 추가!
add_user_message(messages, "그 응용 분야는 무엇인가요?")
follow_up = chat(messages)             # 전체 기록 포함하여 전송
⚠️

흔한 실수: Claude 응답을 메시지 기록에 추가하지 않으면 다음 요청에서 context가 사라집니다. add_assistant_message(messages, answer)를 항상 호출하세요.

스트리밍 응답

스트리밍을 사용하면 생성이 완료되기 전에 토큰이 도착하는 즉시 사용자에게 보여줄 수 있습니다.

# 스트리밍으로 실시간 출력
with client.messages.stream(
    model=model,
    max_tokens=1000,
    messages=[{"role": "user", "content": "파이썬의 장점을 설명해주세요"}]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)  # 실시간 출력

# 스트림 완료 후 최종 메시지 접근 가능
final_message = stream.get_final_message()

구조화된 출력 (JSON)

데이터 추출이나 구조적 응답이 필요할 때 JSON 형식을 강제할 수 있습니다.

response = client.messages.create(
    model=model,
    max_tokens=500,
    messages=[{
        "role": "user",
        "content": """다음 텍스트에서 정보를 추출하세요:
        '홍길동, 28세, 서울 거주, 개발자'

        JSON 형식으로만 응답하세요:
        {"name": "...", "age": ..., "city": "...", "job": "..."}"""
    }]
)

import json
data = json.loads(response.content[0].text)
print(data["name"])  # "홍길동"
💡

보다 안정적인 구조화 출력이 필요하다면 Tool Use(패널 06)를 사용하세요. Tool Use는 JSON Schema로 출력 형식을 강제할 수 있어 파싱 오류가 훨씬 줄어듭니다.

03 · SYSTEM PROMPTS

시스템 프롬프트 & 출력 제어

시스템 프롬프트로 Claude의 역할과 행동을 정의하고, Temperature와 기타 파라미터로 출력을 세밀하게 제어합니다.

이 섹션에서 배울 내용

  • 시스템 프롬프트의 역할과 효과적인 작성법
  • Temperature로 창의성 수준 조절하기
  • max_tokens, stop_sequences로 출력 길이 제어
  • 어시스턴트 프리필(Prefill)로 응답 형식 유도

시스템 프롬프트

시스템 프롬프트는 Claude에게 역할, 톤, 제약사항을 정의합니다. messages와 독립적으로 system 파라미터로 전달됩니다.

# 수학 튜터 시스템 프롬프트 예시
system_prompt = """당신은 인내심 있는 수학 튜터입니다.
학생의 질문에 직접 답을 주지 마세요.
대신 힌트와 유도 질문으로 학생이 스스로 답을 찾도록 도와주세요.
단계적으로 안내하고, 격려하는 톤을 유지하세요."""

messages = []
add_user_message(messages, "5x + 2 = 3 에서 x를 구하려면 어떻게 해야 하나요?")
answer = chat(messages, system=system_prompt)
💡

시스템 프롬프트는 대화 전반에 걸쳐 일관되게 적용됩니다. 멀티턴 대화에서도 system 파라미터를 매번 전달해야 Claude가 역할을 유지합니다.

Temperature — 창의성 다이얼

Temperature는 Claude 응답의 예측 가능성과 창의성을 조절하는 0~1 사이의 값입니다.

0.0 ~ 0.3
정밀 모드
  • 사실적 응답
  • 코딩 지원
  • 데이터 추출
  • 콘텐츠 검열
0.4 ~ 0.7
균형 모드
  • 요약 작업
  • 교육 콘텐츠
  • 문제 해결
  • 제한된 창작
0.8 ~ 1.0
창의 모드
  • 브레인스토밍
  • 자유 창작
  • 마케팅 문안
  • 유머/농담
# Temperature 제어 예시
def chat(messages, system=None, temperature=1.0):
    params = {
        "model": model,
        "max_tokens": 1000,
        "messages": messages,
        "temperature": temperature,
    }
    if system:
        params["system"] = system
    message = client.messages.create(**params)
    return message.content[0].text

# 예측 가능한 코드 생성 (temperature 낮게)
code = chat(messages, temperature=0.1)

# 창의적인 슬로건 생성 (temperature 높게)
slogans = chat(messages, temperature=0.9)

출력 길이 제어 & Stop Sequences

response = client.messages.create(
    model=model,
    max_tokens=500,        # 최대 500 토큰 (예산이지 목표가 아님)
    stop_sequences=["---", "END"],  # 이 문자열이 등장하면 즉시 중단
    messages=[{"role": "user", "content": "항목별 요약을 작성해주세요"}]
)

어시스턴트 프리필 (Prefill)

응답의 시작 부분을 미리 지정하여 원하는 형식으로 유도할 수 있습니다.

messages = [
    {"role": "user", "content": "파이썬과 자바스크립트를 비교해주세요"},
    {"role": "assistant", "content": "```json\n{"}  # ← JSON으로 시작하도록 유도
]

response = client.messages.create(model=model, max_tokens=1000, messages=messages)
04 · PROMPT EVALS

프롬프트 평가 (Evals)

체계적인 평가 워크플로우로 프롬프트 품질을 측정하고 지속적으로 개선합니다.

이 섹션에서 배울 내용

  • Eval이 필요한 이유 — "LLM 심리학" 극복하기
  • 5단계 Eval 워크플로우 전체 흐름
  • Haiku로 테스트 데이터셋 자동 생성하기
  • 코드 기반 채점 vs. 모델 기반 채점 비교
  • 평가 결과 분석 및 프롬프트 반복 개선

핵심 요약

핵심 원칙
느낌이 아닌 데이터로 프롬프트 결정 — 체계적 측정이 곧 품질
채점 방식
코드 기반(정확 매칭) + 모델 기반(의미적 판단) 조합
데이터셋
Haiku로 빠르게 생성 — 저비용으로 다양한 테스트 케이스 확보

5단계 Eval 워크플로우

1

작업 정의

Claude에게 맡길 정확한 작업과 성공 기준을 명확히 정의합니다.

2

테스트 데이터셋 생성

실제 입력 예시와 기대 출력(golden answers)을 포함한 데이터셋을 준비합니다.

3

Eval 실행

각 테스트 케이스에 대해 Claude를 실행하고 출력을 수집합니다.

4

채점

코드 기반 또는 모델 기반으로 각 응답을 점수화합니다.

5

반복 개선

결과를 분석하고 프롬프트를 수정한 뒤 다시 Eval을 실행합니다.

Haiku로 테스트 데이터셋 생성

# 빠르고 저렴한 claude-haiku로 테스트 케이스 생성
haiku_model = "claude-haiku-4-5@20251001"

def generate_test_cases(task_description, n=20):
    prompt = f"""다음 작업에 대한 테스트 케이스 {n}개를 JSON 형식으로 생성하세요.

작업: {task_description}

형식:
[
  {{"input": "...", "expected_output": "...", "category": "..."}},
  ...
]"""

    response = client.messages.create(
        model=haiku_model,
        max_tokens=4000,
        messages=[{"role": "user", "content": prompt}]
    )
    return json.loads(response.content[0].text)

test_cases = generate_test_cases("고객 리뷰에서 감정(긍정/부정/중립)을 분류하기")

모델 기반 채점 (LLM-as-Judge)

def grade_response(question, expected, actual):
    prompt = f"""다음 답변이 기대 답변과 의미적으로 동일한지 평가하세요.

질문: {question}
기대 답변: {expected}
실제 답변: {actual}

"correct" 또는 "incorrect"로만 응답하세요."""

    response = client.messages.create(
        model=haiku_model,
        max_tokens=10,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text.strip().lower() == "correct"

# 전체 Eval 실행
results = []
for case in test_cases:
    response = chat([{"role": "user", "content": case["input"]}])
    is_correct = grade_response(case["input"], case["expected_output"], response)
    results.append(is_correct)

accuracy = sum(results) / len(results)
print(f"정확도: {accuracy:.1%}")
05 · PROMPT ENGINEERING

프롬프트 엔지니어링

Claude로부터 더 정확하고 유용한 응답을 끌어내는 4가지 핵심 기법을 익힙니다.

이 섹션에서 배울 내용

  • 명확성 & 직접성 — 구체적으로 원하는 것을 명시
  • 구체성 — 모호함 없이 상세하게 지시하기
  • XML 태그로 프롬프트 구조화하기
  • 예시 제공 (Few-shot) 으로 품질 향상
기법 1
명확성 & 직접성
원하는 것을 명시적으로 요청하세요. Claude는 의도를 추측하지 않습니다.
기법 2
구체성
길이, 형식, 대상 독자, 톤 등 세부 사항을 구체적으로 지정하세요.
기법 3
XML 태그
복잡한 프롬프트는 XML 태그로 구조화하여 Claude가 각 부분을 명확히 이해하도록 합니다.
기법 4
예시 제공
기대하는 입출력 예시를 보여주면 Claude가 패턴을 학습하여 더 정확한 응답을 생성합니다.

기법 비교: 나쁜 예 vs. 좋은 예

기법나쁜 프롬프트좋은 프롬프트
명확성"이메일 써줘""고객 환불 거절에 대한 정중한 사과 이메일을 써줘"
구체성"설명해줘""10살 아이가 이해할 수 있는 수준으로, 3문장 이내로 설명해줘"
XML긴 텍스트를 하나의 덩어리로<document>, <task>, <format> 태그로 분리
예시"감정을 분류해줘""긍정: '좋아요!' / 부정: '별로야' / 중립: '그냥 그래' 처럼 분류해줘"

prompt = """

06 · TOOL USE

Tool Use (도구 사용)

Claude에게 함수 호출 능력을 부여하여 실시간 데이터 접근, 계산, 외부 서비스 연동을 가능하게 합니다.

이 섹션에서 배울 내용

  • Tool Use의 작동 원리 — 4단계 사이클
  • JSON Schema로 도구 함수 정의하기
  • tool_use 응답 감지 및 함수 실행
  • tool_result를 Claude에게 다시 전송하기
  • 멀티플 도구 & Batch Tool Use
  • toolChoice로 도구 사용 강제/방지

핵심 요약

핵심 원리
Claude는 도구를 직접 실행하지 않음 — 어떤 도구를 어떤 인수로 호출할지 "결정"만 함
흐름
도구 정의 → API 요청 → tool_use 감지 → 실행 → tool_result 전송 → 최종 응답

Tool Use 4단계 사이클

1

도구 정의

JSON Schema로 도구의 이름, 설명, 입력 파라미터를 정의합니다.

2

API 요청

tools 파라미터에 도구 목록을 포함하여 Claude에게 요청합니다.

3

도구 실행

stop_reason == "tool_use"이면 여러분의 코드가 해당 함수를 실행합니다.

4

결과 전송

tool_result 메시지로 실행 결과를 Claude에게 보내 최종 응답을 생성하게 합니다.

전체 Tool Use 구현 예시

import json

# 1. 도구 정의 (JSON Schema)
tools = [{
    "name": "get_weather",
    "description": "특정 도시의 현재 날씨를 가져옵니다",
    "input_schema": {
        "type": "object",
        "properties": {
            "city": {"type": "string", "description": "도시 이름"},
            "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
        },
        "required": ["city"]
    }
}]

# 실제 함수 (여러분이 구현)
def get_weather(city, unit="celsius"):
    return {"city": city, "temperature": 22, "condition": "맑음"}

# 2. API 요청
messages = [{"role": "user", "content": "서울 날씨 어때?"}]
response = client.messages.create(
    model=model, max_tokens=1000, messages=messages, tools=tools
)

# 3. tool_use 감지 & 실행
while response.stop_reason == "tool_use":
    tool_uses = [b for b in response.content if b.type == "tool_use"]

    messages.append({"role": "assistant", "content": response.content})
    tool_results = []

    for tool_use in tool_uses:
        result = get_weather(**tool_use.input)
        tool_results.append({
            "type": "tool_result",
            "tool_use_id": tool_use.id,
            "content": json.dumps(result, ensure_ascii=False)
        })

    # 4. 결과 전송 → 최종 응답
    messages.append({"role": "user", "content": tool_results})
    response = client.messages.create(
        model=model, max_tokens=1000, messages=messages, tools=tools
    )

print(response.content[0].text)  # "서울은 현재 22도이며 맑습니다."

07 · RAG

RAG (검색 증강 생성)

외부 문서나 데이터베이스에서 관련 정보를 검색하여 Claude에게 컨텍스트로 제공하는 RAG 파이프라인을 구축합니다.

이 섹션에서 배울 내용

  • RAG가 필요한 이유 — 지식 컷오프 및 프라이빗 데이터
  • 텍스트 청킹 전략 (고정 크기 vs. 의미 기반)
  • VoyageAI 임베딩으로 벡터화하기
  • 코사인 유사도 기반 벡터 검색
  • BM25 렉시컬 검색으로 하이브리드 검색
  • Contextual Retrieval로 정확도 향상
  • Vertex AI Search를 활용한 관리형 RAG

RAG 파이프라인 전체 흐름

1

문서 청킹

긴 문서를 적절한 크기(500~1000 토큰)의 청크로 분할합니다. 오버랩(10~20%)으로 경계 정보 손실을 방지합니다.

2

임베딩 생성 & 저장

각 청크를 임베딩 벡터로 변환하고 벡터 DB에 저장합니다.

3

쿼리 임베딩

사용자 질문도 같은 모델로 임베딩합니다.

4

유사도 검색

코사인 유사도로 가장 관련성 높은 상위 K개 청크를 검색합니다.

5

Claude 응답 생성

검색된 청크를 컨텍스트로 Claude에게 전달하여 답변을 생성합니다.

기본 RAG 구현

import voyageai
import numpy as np

voyage_client = voyageai.Client()

def generate_embedding(text):
    result = voyage_client.embed([text], model="voyage-3")
    return result.embeddings[0]

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 문서 청킹 & 임베딩
chunks = chunk_document(document, chunk_size=500, overlap=50)
vector_store = []
for chunk in chunks:
    embedding = generate_embedding(chunk)
    vector_store.append({"content": chunk, "embedding": embedding})

# 검색 & 응답 생성
def rag_query(question, k=3):
    q_embedding = generate_embedding(question)

    # 유사도 계산 & 상위 K개 선택
    scored = [(cosine_similarity(q_embedding, doc["embedding"]), doc["content"])
              for doc in vector_store]
    top_k = sorted(scored, reverse=True)[:k]
    context = "\n\n".join([content for _, content in top_k])

    # Claude에 컨텍스트와 함께 질문
    prompt = f"""<context>
{context}
</context>

위 컨텍스트를 바탕으로 질문에 답하세요: {question}"""

    response = client.messages.create(
        model=model, max_tokens=1000,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text

하이브리드 검색 (시맨틱 + BM25)

시맨틱 검색만으로는 특정 용어(예: INC-2023-Q4-011)를 정확히 찾지 못할 수 있습니다. BM25 렉시컬 검색과 결합하면 두 가지 장점을 모두 얻을 수 있습니다.

방식장점단점
시맨틱 검색의미적으로 유사한 내용 검색정확한 키워드 매칭 부족
BM25 렉시컬특정 용어나 ID 정확 매칭동의어나 문맥 이해 부족
하이브리드두 방식의 장점 결합구현 복잡도 증가
☁️

Vertex AI Search: GCP의 관리형 검색 서비스로, 직접 벡터 DB를 구축하지 않고도 엔터프라이즈급 RAG를 구현할 수 있습니다. Google Cloud Storage나 BigQuery의 데이터를 자동으로 인덱싱하고 검색합니다.

MCP (모델 컨텍스트 프로토콜)

MCP로 Claude에게 표준화된 방식으로 도구와 리소스를 연결하여 진정한 AI 에이전트를 구축합니다.

이 섹션에서 배울 내용

  • MCP 서버에서 도구(Tools) 정의하기

  • Python MCP 서버 구현 패턴
  • Vertex AI 환경에서 MCP 클라이언트 구현

핵심 요약

MCP란?
Model Context Protocol — AI 모델과 외부 시스템을 연결하는 표준화된 오픈 프로토콜
3가지 기능
Tools (함수 실행) + Resources (데이터 접근) + Prompts (재사용 가능한 템플릿)
GCP 통합
BigQuery, Cloud Storage, Pub/Sub 등 GCP 서비스용 MCP 서버 활용 가능

MCP 서버 구현 (Python)

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("GCP 데이터 서버")

# Tool 정의
@mcp.tool()
def query_bigquery(sql: str) -> str:
    """BigQuery에서 SQL 쿼리를 실행합니다."""
    from google.cloud import bigquery
    client = bigquery.Client()
    results = client.query(sql).result()
    return str(list(results))

@mcp.tool()
def read_gcs_file(bucket: str, path: str) -> str:
    """Google Cloud Storage에서 파일을 읽습니다."""
    from google.cloud import storage
    client = storage.Client()
    bucket_obj = client.bucket(bucket)
    blob = bucket_obj.blob(path)
    return blob.download_as_text()

# Resource 정의
@mcp.resource("gcp://datasets/list")
def list_datasets() -> str:
    """사용 가능한 BigQuery 데이터셋 목록을 반환합니다."""
    return "dataset_a, dataset_b, dataset_c"

if __name__ == "__main__":
    mcp.run()

MCP 클라이언트 구현 (Vertex AI)

from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client
from anthropic import AnthropicVertex

client = AnthropicVertex(region="global", project_id="my-project")
model = "claude-sonnet-4@20250514"

async def run_with_mcp():
    server_params = StdioServerParameters(
        command="python", args=["gcp_server.py"]
    )

    async with stdio_client(server_params) as (read, write):
        async with ClientSession(read, write) as session:
            await session.initialize()

            # 사용 가능한 도구 목록 조회
            tools_result = await session.list_tools()
            tools = [convert_to_claude_tool(t) for t in tools_result.tools]

            # Claude에게 GCP 도구와 함께 질문
            response = client.messages.create(
                model=model,
                max_tokens=4000,
                tools=tools,
                messages=[{"role": "user",
                           "content": "sales 데이터셋에서 이번달 매출 합계를 조회해줘"}]
            )

핵심 요약

에이전트란?
목표가 주어지면 스스로 계획하고 도구를 사용하며 완료될 때까지 반복하는 AI 시스템
핵심 속성
인식(Perception) + 추론(Reasoning) + 행동(Action)
GCP 통합
Vertex AI Agents로 완전 관리형 에이전트 인프라 운영

에이전트 루프 구현

에이전트는 목표가 달성될 때까지 생각 → 도구 사용 → 결과 처리를 반복합니다.

from anthropic import AnthropicVertex

client = AnthropicVertex(region="global", project_id="my-project")
model = "claude-sonnet-4@20250514"

def run_agent(goal: str, tools: list, tool_functions: dict):
    messages = [{"role": "user", "content": goal}]

    while True:
        response = client.messages.create(
            model=model,
            max_tokens=4000,
            tools=tools,
            messages=messages,
            system="""당신은 목표를 달성하기 위해 도구를 활용하는 에이전트입니다.
            주어진 도구를 최대한 활용하여 목표를 달성하세요."""
        )

        # 에이전트 완료 — 자연스러운 종료
        if response.stop_reason == "end_turn":
            return response.content[0].text

        # 도구 사용 처리
        if response.stop_reason == "tool_use":
            messages.append({"role": "assistant", "content": response.content})
            tool_results = []

            for block in response.content:
                if block.type == "tool_use":
                    fn = tool_functions[block.name]
                    result = fn(**block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": str(result)
                    })

            messages.append({"role": "user", "content": tool_results})

# 에이전트 실행
result = run_agent(
    goal="BigQuery에서 지난 30일 매출 데이터를 분석하고 인사이트를 제공해줘",
    tools=gcp_tools,
    tool_functions={"query_bigquery": query_bigquery, "read_gcs_file": read_gcs_file}
)

에이전트의 3가지 핵심 속성

속성 1
인식 (Perception)
텍스트, 이미지, 데이터, 도구 결과 등 다양한 입력을 이해하고 상황을 파악합니다.
속성 2
추론 (Reasoning)
현재 상황을 분석하고, 목표 달성을 위한 최적 행동을 계획하며, 결과를 평가합니다.
속성 3
행동 (Action)
도구 호출, 코드 실행, 파일 조작, 외부 API 호출 등 실제 세계에 영향을 미칩니다.

Claude Code 활용

Claude Code는 코드베이스 전체를 이해하고 수정할 수 있는 AI 코딩 어시스턴트입니다.

설치 및 시작

# Claude Code 설치
$ npm install -g @anthropic-ai/claude-code

# 프로젝트 디렉토리에서 시작
$ cd my-vertex-project
$ claude

# 주요 명령어
"Vertex AI 클라이언트 설정 파일을 만들어줘"
"BigQuery 연결 오류를 디버깅해줘"
"이 RAG 파이프라인의 성능을 개선해줘"
"GCP 서비스 계정 인증 코드를 리팩토링해줘"

멀티 에이전트로 Claude Code 활용

# Claude Code를 서브에이전트로 실행 (프로그래밍 방식)
import subprocess

result = subprocess.run(
    ["claude", "-p", "tests/를 실행하고 실패한 테스트를 모두 수정해줘", "--output-format", "json"],
    capture_output=True, text=True
)
print(result.stdout)

Vertex AI Agents (관리형 에이전트)

Google Cloud의 Vertex AI Agents 서비스는 에이전트 인프라를 완전히 관리해줍니다.

방식장점적합한 상황
직접 구현 (에이전트 루프)완전한 제어권, 유연성특수한 에이전트 로직이 필요한 경우
Vertex AI Agents인프라 자동 관리, GCP 통합빠른 배포, GCP 네이티브 통합
MCP + Vertex표준화된 도구 연결다양한 GCP 서비스 통합
⚠️

에이전트 안전성: 에이전트가 BigQuery 수정, GCS 파일 삭제, Pub/Sub 메시지 발행 등 되돌리기 어려운 작업을 수행할 때는 반드시 인간 확인 체크포인트를 포함하세요. IAM 최소 권한 원칙을 적용하여 에이전트의 권한을 필요한 최소한으로 제한합니다.

최종 정리

이 코스에서 배운 모든 것을 종합하면: AnthropicVertex 클라이언트로 시작하고, 멀티턴 대화 + 시스템 프롬프트로 기반을 다지고, Evals로 품질을 측정하고, Tool Use + RAG로 기능을 확장하고, MCP로 GCP 서비스를 연결하고, 최종적으로 에이전트 루프로 자율적인 시스템을 구축합니다. Vertex AI의 엔터프라이즈 인프라 위에서 Claude의 강력한 AI 능력을 최대한 활용하세요.
