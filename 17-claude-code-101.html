LESSON 1

What is Claude Code?

원본: https://anthropic.skilljar.com/claude-code-101/469788
WHAT YOU'LL LEARN
  • Claude Code가 일반 AI 채팅과 다른 점
  • AI 에이전트 개념과 Claude Code의 에이전트 구조
  • Claude Code가 실제로 할 수 있는 일들

Claude Code란?

Claude Code는 코드베이스를 이해하고, 파일을 편집하고, 명령을 실행하고, 기존 개발자 툴과 연동해 더 빠르게 작업을 완료할 수 있도록 돕는 에이전틱 코딩 도구입니다. 터미널, Visual Studio Code, Claude Desktop 앱, 웹, JetBrains IDE에서 사용할 수 있습니다.

Claude.ai와 무엇이 다른가?

Claude.ai를 사용해본 적 있다면, Claude Code가 무엇이 다른지 궁금할 것입니다. Claude.ai와 달리, Claude Code는 파일, 터미널, 전체 코드베이스에 직접 접근할 수 있습니다. 코드를 복사해서 붙여넣는 대신, 직접 들어가서 작업을 수행합니다.

핵심 차이점은 Claude Code가 AI 에이전트로 동작한다는 것입니다.

AI 에이전트란?

AI 에이전트는 환경과 상호작용하고 정의된 목표를 달성하기 위한 행동을 수행할 수 있는 소프트웨어입니다. 핵심적으로, 대형 언어 모델이 실시간으로 루프 안에서 작동합니다. AI 에이전트는 목표 달성을 위해 도구, 외부 서비스, 심지어 다른 AI 에이전트에도 접근할 수 있습니다.

Claude Code가 실제로 할 수 있는 일

  • ▸ 코드베이스 읽기 및 이해: 기능을 설명하거나 코드 전체에서 버그를 추적할 수 있습니다
  • ▸ 프로젝트 파일 편집: 함수를 리팩토링하고 참조하는 모든 파일을 업데이트할 수 있습니다
  • ▸ 터미널 명령 실행: 빌드 스크립트를 실행하고, 테스트를 돌리고, 패키지를 설치하고, 결과를 보고 다음 행동을 결정합니다
  • ▸ 웹 검색: 문서나 최신 API 레퍼런스가 필요하면 직접 검색합니다

Claude Code를 효과적으로 사용하기

세 가지 개념을 항상 염두에 두세요:

1. 컨텍스트 윈도우
Claude의 작업 메모리입니다. 많은 내용을 담을 수 있지만 전부는 아닙니다. 여기서 '에이전틱' 측면이 드러납니다 — Claude는 전체를 컨텍스트에 로드하지 않고도 코드베이스에서 답을 찾는 전략적인 방법을 찾아냅니다.
2. 권한 요청
기본적으로 Claude Code는 명령 실행이나 변경 전에 허가를 요청합니다. 세밀하게 관여하든 방임하든, 항상 제어권은 여러분에게 있습니다.
3. 실수 가능성
어떤 도구와 마찬가지로, Claude Code도 완벽하지 않습니다. 의도를 잘못 이해하거나, 버그를 도입하거나, 과도하게 설계할 수 있습니다. 관여를 유지하면 조기에 발견할 수 있습니다.
RECAP
  • Claude Code는 파일을 읽고, 명령을 실행하고, 외부 도구에 연결하는 에이전틱 코딩 도구입니다
  • 터미널, VS Code, JetBrains, Claude Desktop 앱에서 사용 가능합니다
  • AI 에이전트로서 루프 안에서 동작하며, 결과를 검증하고 목표를 향해 반복합니다
LESSON 2

How Claude Code works

원본: https://anthropic.skilljar.com/claude-code-101/469789
WHAT YOU'LL LEARN
  • 에이전틱 루프의 동작 방식
  • 컨텍스트 윈도우 관리
  • 도구(Tools)의 역할
  • 권한 모드 종류

에이전틱 루프

Claude Code는 에이전틱 루프로 동작합니다:

  1. 1.프롬프트를 입력합니다
  2. 2.Claude는 모델과 상호작용하며 필요한 컨텍스트를 수집합니다 — 텍스트 또는 실행 가능한 툴 호출을 반환합니다
  3. 3.파일 편집이나 명령 실행 등 행동을 취합니다
  4. 4.결과를 검증하고 프롬프트가 목표를 달성했는지 판단합니다
  5. 5.달성했으면 완료하고 다음 프롬프트를 기다립니다. 그렇지 않으면 완료될 때까지 루프를 반복합니다

이 루프 내내 컨텍스트를 추가하거나, 중단하거나, 방향을 조정해 목표를 향해 유도할 수 있습니다.

컨텍스트

Claude는 대화, 파일 내용, 명령 출력 등을 저장하고 참조할 수 있는 컨텍스트 윈도우를 가집니다. 한계에 도달하면 Claude Code는 대화를 자동으로 압축합니다 — 컨텍스트 윈도우를 사용 가능한 크기로 줄이기 위해 제거하거나 요약할 수 있는 것을 자동으로 결정합니다.

도구(Tools)

도구는 에이전트가 동작하는 근간입니다. 대부분의 AI 어시스턴트는 단순히 텍스트를 받아 텍스트를 반환합니다. 도구는 Claude Code가 작업 완료에 가까워지기 위해 코드를 언제 실행할지 결정할 수 있게 합니다. 파일 읽기 도구, 웹 검색 도구 등 다양한 기능이 있습니다. Claude Code는 의미론적 이해를 통해 언제 도구를 호출하고 결과를 어떻게 활용할지 결정합니다.

권한 모드

Claude Code에는 여러 권한 모드가 있습니다:

기본 동작
파일 편집이나 셸 명령 실행 전 명시적 허가를 요청합니다
자동 수락
파일은 묻지 않고 편집되지만, 명령은 여전히 승인이 필요합니다
플랜 모드
읽기 전용 도구를 사용해 작업 시작 전 실행 계획을 수립합니다
⚠️ 주의
권한을 건너뛸 때는 주의하세요 — Claude Code가 명령을 자유롭게 실행하게 하면 실수가 발생하기 전에 잡기 어려울 수 있습니다.
RECAP
  • Claude Code는 에이전틱 루프, 관리된 컨텍스트 윈도우, 도구, 설정 가능한 권한 모드를 결합합니다
  • 코드베이스를 읽고, 행동을 취하고, 자체 작업을 검증할 수 있습니다
  • 이것이 채팅 창과 근본적으로 다른 점입니다
LESSON 3

Installing Claude Code

원본: https://anthropic.skilljar.com/claude-code-101/469790
WHAT YOU'LL LEARN
  • 터미널, VS Code, JetBrains, Desktop, Web에서의 설치 방법
  • 각 플랫폼별 특징과 장단점
  • 어떤 환경을 선택할지 기준

터미널

macOS, Linux, WSL에서는 curl 명령으로 한 번에 설치할 수 있습니다. Homebrew도 사용할 수 있지만 자동 업데이트를 지원하지 않습니다.

Windows에서는 PowerShell에서 Invoke-RestMethod 명령, CMD에서 curl 명령, 또는 winget 명령을 사용할 수 있습니다 (winget도 자동 업데이트 미지원).

설치 후 claude 명령을 실행할 수 있어야 합니다. 실행이 안 되면 터미널을 재시작하세요. 프로젝트 디렉토리로 이동해 실행하면 됩니다:

claude

색상 테마 선택, Claude 계정(Pro, Max, Enterprise) 로그인 또는 API 키 입력 등 초기 설정을 진행합니다. claude를 실행한 디렉토리와 모든 하위 폴더에 접근 권한이 부여됩니다.

Visual Studio Code

확장 프로그램 패널을 열고 "Claude Code"를 검색합니다. 파란색 인증 마크가 있는 Anthropic의 확장을 찾아 설치합니다.

설치 후 VS Code를 재시작해야 할 수 있습니다. 실행 후 Ctrl/Cmd + Shift + P로 명령 팔레트를 열고 "Claude Code Open in New Tab"을 검색합니다. 사이드바에 Claude 로고가 보이면 클릭해도 됩니다.

JetBrains

JetBrains Marketplace에서 Claude Code 플러그인을 설치합니다. 설치 후 IDE를 재시작하면 Claude 로고가 보입니다. 클릭하면 에디터 옆에 터미널 경험을 제공하는 패널이 열립니다.

Desktop

Claude Desktop을 설치하고 로그인하면 상단에 "Code" 토글이 보입니다. 채팅과 비슷하지만 특정 폴더에서 작업하고, 권한을 변경하고, 클라우드 환경에서도 작업할 수 있습니다.

Web

웹에서는 claude.ai/code에 접속하거나 채팅 앱 사이드바의 "Code" 탭을 클릭합니다. Desktop 앱과 유사하지만 GitHub 저장소로 제한됩니다.

어떤 것을 선택해야 할까요?
  • 최신 기능을 원한다면 터미널 — 기능이 가장 먼저 배포됩니다
  • IDE 통합을 원한다면 VS Code 또는 JetBrains 확장
  • Claude가 백그라운드에서 실행되길 원한다면 Desktop 앱
  • GitHub 저장소 원격 작업을 원한다면 Web
RECAP
  • Claude Code는 터미널, VS Code, JetBrains, Desktop, Web 5가지 환경에서 사용 가능합니다
  • 터미널이 항상 최신 기능을 먼저 받습니다
  • 어떤 환경을 선택하든 기본 경험은 동일합니다
LESSON 4

Your first prompt

원본: https://anthropic.skilljar.com/claude-code-101/469791
WHAT YOU'LL LEARN
  • 자동 수락 vs 승인 모드 전환 방법
  • 플랜 모드의 활용법
  • 효과적인 첫 프롬프트 작성법

자동 수락 vs 승인 모드

Claude가 매번 파일 변경을 자동 수락할지, 아니면 명시적 허가를 요청할지 선택할 수 있습니다. Shift + Tab을 눌러 모드를 전환합니다.

승인 모드
Claude가 파일을 편집하거나 명령을 실행할 때마다 허가를 요청합니다
자동 수락 모드
파일 편집은 자동으로 승인되지만, 명령은 여전히 허가가 필요합니다

플랜 모드

Shift + Tab 메뉴에 플랜 모드가 있습니다. 플랜 모드는 프롬프트를 받아 읽기 전용 도구로 코드베이스를 분석하고 구현 방법을 연구합니다. 진행하면서 명확화 질문을 하고, 실행할 수 있는 상세한 계획을 반환합니다.

플랜 모드는 복잡한 변경을 계획하거나 안전한 코드 리뷰를 하는 데 최적입니다.

예시: 다크 모드 토글 추가

실습 예시

다크 모드 토글이 필요한 앱이 있다고 가정합니다. 프로젝트 루트 디렉토리를 열고 claude를 실행합니다. Shift + Tab을 몇 번 눌러 플랜 모드에 진입한 후 다음과 같은 프롬프트를 작성합니다:

"앱 전체에 다크 모드를 구현해야 합니다. 기존 라이트 테마를 기반으로 좋은 대비 색상을 찾아 헤더에 라이트/다크 모드를 전환할 수 있는 토글 스위치를 만들어줄 수 있나요?"

Claude가 계획을 세우도록 기다립니다. 계획이 괜찮아 보이면 수락하고 Claude가 각 단계별로 승인을 요청하도록 합니다.

RECAP
  • Shift + Tab으로 승인 모드, 자동 수락 모드, 플랜 모드를 전환합니다
  • 프롬프트는 최대한 구체적으로 작성하세요
  • 플랜 모드는 코드 작성 전에 Claude가 세부 사항을 깊이 파악할 수 있게 합니다
LESSON 5

The explore → plan → code → commit workflow

원본: https://anthropic.skilljar.com/claude-code-101/469792
WHAT YOU'LL LEARN
  • Explore → Plan → Code → Commit 워크플로우의 각 단계
  • 플랜 모드를 활용한 탐색과 계획
  • 코딩 단계를 원활하게 하는 팁
  • 코드 리뷰와 커밋 자동화

Explore → Plan → Code → Commit 워크플로우

이 강의에서 한 가지만 가져간다면 바로 이 워크플로우입니다: Explore, Plan, Code, Commit. 이 없이는 대부분의 사람들이 Claude에게 바로 코드 작성을 요청합니다 — 이는 나중에 더 많은 수정을 의미합니다.

탐색과 계획 (Explore & Plan)

가장 빠른 방법은 플랜 모드를 사용하는 것입니다. 플랜 모드에서 Claude는 파일을 편집할 수 없습니다 — 구현 방법을 파악하기 위해 파일을 읽기만 합니다.

Shift + Tab을 눌러 플랜 모드에 진입한 후 예시 프롬프트:

예시 프롬프트
"이미지 업로드 파이프라인에 WebP 변환을 추가해야 합니다. 파이프라인의 어디에 들어가야 할지, 새로운 의존성이 필요한지, 어떻게 접근할지 파악해주세요."

Claude가 관련 파일을 읽고, 웹 검색을 하고, 행동 계획을 제시합니다. 검토 후 기준에 맞는지 결정합니다. 맞지 않으면 특정 부분 수정을 요청합니다.

코딩 단계 (Code)

계획이 좋아 보이면 "승인"을 선택해 Claude가 목록 항목들을 처리하도록 합니다.

코딩 단계를 원활하게 하는 팁
  • 성공 기준 정의: 결과가 "올바른지" 명확히 해야 Claude가 확신을 가질 수 있습니다
  • 도구 추가: Claude가 목표를 달성하는 데 도움이 되는 도구는 많은 시간을 절약합니다 (예: 웹 UI 작업 시 Claude in Chrome 확장)
  • 테스트 스위트 포함: Claude가 지속적으로 검증할 수 있는 테스트를 제공합니다

커밋 단계 (Commit)

변경 사항을 직접 테스트하고 만족하면 코드를 푸시할 차례입니다. 커밋 전에 서브에이전트 코드 리뷰어로 작업을 검토합니다. 서브에이전트는 세션에서 메인 에이전트가 가질 수 있는 편향 없이 코드베이스를 새로운 시각으로 바라봅니다.

RECAP
  • Explore: Claude에게 프로젝트의 관련 컨텍스트를 제공합니다
  • Plan: Claude가 성공 측정에 사용할 행동 계획을 수립합니다
  • Code: 최종 결과를 결정하기 전 여러분과 Claude 사이의 반복 과정입니다
  • Commit: 코드를 검토하고 푸시해 다음 기능을 시작합니다
LESSON 6

Context management

원본: https://anthropic.skilljar.com/claude-code-101/469793
WHAT YOU'LL LEARN
  • 컨텍스트 윈도우가 무엇인지와 왜 중요한지
  • 컨텍스트 관리 명령어 (/compact, /clear, /context)
  • 컨텍스트를 절약하는 실용적인 팁

컨텍스트 윈도우란?

컨텍스트는 Claude의 작업 메모리입니다. 프롬프트를 입력하거나, 파일을 읽거나, 도구 호출을 실행하거나, 결과를 받을 때마다 모두 컨텍스트 윈도우 공간을 차지합니다. 공간이 제한적이므로 효율적으로 활용하는 것이 중요합니다.

컨텍스트가 가득 찰 때

한계에 가까워지면 컨텍스트 윈도우가 자동으로 압축됩니다. 압축은 중요한 세부 사항을 요약하고 불필요한 도구 호출 결과를 제거해 공간을 확보합니다. 이 과정에서 세부 사항이 손실될 수 있다는 점에 주의하세요.

컨텍스트 관리 명령어

명령어 설명 언제 사용
/compact 현재까지의 내용을 압축 현재 기능 작업 중 컨텍스트 한계에 다가갈 때
/clear 모든 기록 완전 삭제 새 기능 작업 시작 시
/context 현재 컨텍스트 상태 확인 현재 컨텍스트 사용 현황 파악

컨텍스트 공간 절약 팁

구체적으로 작성하기
막연한 프롬프트는 오히려 컨텍스트를 더 많이 소비합니다. 명확한 지시 없이 Claude는 코드베이스를 더 많이 탐색하고 자체 추론을 해야 합니다
MCP 서버 관리
MCP 서버는 기본적으로 모든 사용 가능한 도구를 컨텍스트에 로드합니다. 현재 프로젝트와 무관한 서버는 비활성화를 고려하세요
서브에이전트 활용
서브에이전트는 메인 에이전트와 별개의 컨텍스트 윈도우에서 실행됩니다. "인증 엔드포인트가 어디 있나요?" 같은 질문은 서브에이전트가 처리하고 요약만 반환합니다
RECAP
  • /compact으로 긴 세션을 요약하고, /clear로 새로 시작합니다
  • 구체적인 프롬프트로 컨텍스트 낭비를 줄이세요
  • 서브에이전트를 활용해 답만 필요한 작업을 위임하세요
LESSON 7

Code review

원본: https://anthropic.skilljar.com/claude-code-101/469794
WHAT YOU'LL LEARN
  • 서브에이전트를 활용한 편향 없는 코드 리뷰
  • /commit-push-pr 스킬로 워크플로우 자동화
  • --from-pr 플래그로 PR 세션 재개

서브에이전트로 코드 리뷰

PR을 푸시하기 전, Claude에게 서브에이전트를 사용해 변경 사항을 검토하도록 요청하세요. 서브에이전트는 자체 컨텍스트 윈도우에서 새로운 시각으로 실행됩니다 — 방금 코드를 작성한 메인 에이전트의 편향이 없습니다.

팁
코드 리뷰어 서브에이전트를 만들 때는 읽기 전용 도구로 제한하세요. 리뷰어는 문제를 발견해야지, 파일을 편집해서는 안 됩니다. 서브에이전트 설정을 저장소에 체크인해 팀 전체가 동일한 리뷰어를 사용하도록 하세요.

/commit-push-pr 스킬

/commit-push-pr 스킬은 커밋, 푸시, PR 생성을 한 단계로 처리합니다. 각 단계를 수동으로 하는 대신, 스킬을 실행하면 Claude가 처리합니다.

자동 알림
Slack MCP 서버가 설정되어 있고 CLAUDE.md에 채널 목록이 있으면, 자동으로 팀 채널에 PR 링크를 게시합니다.

--from-pr로 세션 연결

Claude가 gh pr create를 통해 PR을 생성하면 세션이 해당 PR에 자동으로 연결됩니다. 나중에 돌아와야 할 때 — 리뷰 의견 반영이나 빌드 수정 등 — 다음을 실행합니다:

claude --from-pr <PR_NUMBER>
RECAP
  • 푸시 전 서브에이전트로 편향 없는 코드 리뷰를 받으세요
  • /commit-push-pr로 전체 커밋-PR 흐름을 한 단계로 처리하세요
  • --from-pr으로 나중에 PR 작업을 재개하세요
LESSON 8

The CLAUDE.md file

원본: https://anthropic.skilljar.com/claude-code-101/469795
WHAT YOU'LL LEARN
  • CLAUDE.md 파일이 해결하는 문제
  • CLAUDE.md 파일 구조와 작성법
  • 팀과 개인을 위한 계층적 메모리 시스템

CLAUDE.md 파일이란?

Claude Code의 가장 유용한 기능 중 하나는 CLAUDE.md 파일입니다. Claude Code에게 프로젝트에 대한 지속적인 메모리를 제공합니다.

해결하는 문제

CLAUDE.md 파일 없이 Claude Code를 열면 매번 새로 시작합니다. 코드베이스를 다시 탐색하고, 필요한 의존성을 파악하고, 이미 구현된 기능을 이해해야 합니다. 때로는 가정을 해서 올바른 방향으로 유도하기 어렵게 만듭니다.

CLAUDE.md는 이를 해결합니다. 프로젝트 루트에 추가하는 마크다운 파일로, 세션을 시작할 때마다 Claude Code가 자동으로 읽습니다. 코드베이스의 온보딩 스크립트라고 생각하세요. CLAUDE.md 파일의 내용은 프롬프트에 추가됩니다.

예시 CLAUDE.md

# Project
This is a Next.js 15 app using the App Router, Tailwind, and Drizzle ORM.

# Commands
- Dev server: `pnpm dev`
- Run tests: `pnpm test`
- Lint: `pnpm lint`

# Code Style
- Use 2-space indentation
- Prefer named exports
- All API routes go in app/api/
- Use server actions instead of API routes where possible

팀을 위한 CLAUDE.md

CLAUDE.md를 버전 관리에 커밋해 팀이 혜택을 받도록 하세요. 실제로 대상에 따라 메모리 파일의 계층이 있습니다:

프로젝트 레벨 CLAUDE.md
프로젝트 루트에 위치, 팀과 공유
사용자 레벨 CLAUDE.md
설정 폴더에 위치, 개인용으로 모든 프로젝트에 적용
유용한 팁들
  • 수정 사항을 메모리에 저장: Claude를 반복적으로 수정하게 된다면 (항상 API 라우트 대신 서버 액션을 사용하도록 지시하는 등) Claude에게 해당 규칙을 메모리에 저장하도록 요청하세요
  • 프로젝트 문서 참조: @README.md처럼 @ 기호와 파일 경로를 사용해 문서를 참조할 수 있습니다
  • CLAUDE.md 없이 시작: 지속적으로 모델을 수정해야 하는 곳을 파악한 후 /init을 실행해 Claude가 생성하도록 하세요
RECAP
  • CLAUDE.md는 Claude Code의 지속적인 메모리로, 세션마다 자동으로 읽힙니다
  • 스택, 명령어, 코드 스타일로 시작해 점차 발전시키세요
  • 버전 관리에 커밋해 팀 전체가 활용하도록 하세요
LESSON 9

Subagents

원본: https://anthropic.skilljar.com/claude-code-101/469796
WHAT YOU'LL LEARN
  • 서브에이전트가 컨텍스트 관리를 어떻게 돕는지
  • 서브에이전트 생성 방법
  • 서브에이전트 커스터마이징 옵션

서브에이전트란?

Claude는 작업을 분해하고 구성 요소를 병렬로 실행하는 서브에이전트에 위임할 수 있어 컨텍스트 관리를 향상시킵니다. 각 서브에이전트는 자체 격리된 컨텍스트 윈도우에서 동작합니다.

동작 방식

컨텍스트 윈도우의 많은 부분이 코드베이스를 탐색하는 도구 호출이나 웹 검색 등으로 소비됩니다. Claude가 탐색하는 과정이 현재 개발 중인 기능과 항상 관련이 있는 것은 아닙니다.

여기서 서브에이전트가 등장합니다. Claude가 "이 코드베이스를 탐색해줘"와 같은 작업을 처리하는 서브에이전트를 생성합니다. 서브에이전트는 자체 컨텍스트 윈도우에서 병렬로 실행되고, 완료되면 결과를 요약해 Claude에게 반환합니다.

결과: 전체 탐색 과정 없이 찾던 답만 얻을 수 있습니다.

서브에이전트 생성

서브에이전트는 YAML 프론트매터가 있는 마크다운 파일로 정의됩니다. 가장 쉬운 시작 방법은 Claude가 생성하도록 하는 것입니다:

/agents

"새 에이전트 만들기"를 선택하면 에이전트의 범위 선택, 목적 정의, 접근 가능한 도구 선택, 색상 지정 등 단계를 거칩니다.

추가 커스터마이징

지속적인 메모리
서브에이전트가 대화 간 메모리를 유지할 수 있어 동일한 프로젝트에서 일관성 있게 사용할 때 유용합니다
스킬 사전 로드
skill 키를 추가하고 스킬 이름을 나열해 서브에이전트에 스킬을 사전 로드할 수 있습니다
RECAP
  • 서브에이전트는 자체 컨텍스트 윈도우에서 병렬로 실행됩니다
  • /agents로 새 서브에이전트를 만들 수 있습니다
  • 무거운 탐색 작업을 서브에이전트에 위임하고 메인 컨텍스트를 깔끔하게 유지하세요
  • 더 자세한 내용은 서브에이전트 입문 과정을 참고하세요

LESSON 11

MCP

원본: https://anthropic.skilljar.com/claude-code-101/469797
WHAT YOU'LL LEARN
  • MCP(Model Context Protocol)가 무엇인지
  • MCP 서버 추가 방법과 두 가지 유형
  • 서버 범위 설정 방법
  • 컨텍스트 비용과 최적화

MCP란?

MCP(Model Context Protocol)는 Claude Code가 외부 도구와 데이터 소스에 연결할 수 있는 오픈 표준입니다. 질문을 할 때 Claude는 해당 도구를 언제 사용해야 할지 자동으로 이해합니다.

코드베이스 밖에 많은 컨텍스트가 있습니다 — 데이터베이스, 생산성 앱, 공개 저장소 등. MCP는 이 간격을 연결합니다.

도구(Tools)의 개념

도구는 에이전트에게 작업을 더 효과적으로 완료하는 데 도움이 되는 행동 수행 능력을 제공합니다. 이는 단순히 텍스트 응답만 받는 일반적인 AI와 다릅니다.

예시: 팀이 프로젝트 관리에 Linear를 사용한다면 Linear MCP 서버를 추가해 특정 이슈의 세부 사항을 가져올 수 있습니다. 의존성의 최신 문서가 필요하다면 Context7 같은 docs MCP 서버를 활용할 수 있습니다.

MCP 서버 추가

claude mcp add 명령으로 MCP 서버를 추가할 수 있습니다. 두 가지 주요 유형:

유형 설명
HTTP 서버 원격 서비스용, 서비스 제공자가 호스팅하고 네트워크를 통해 연결
Stdio 서버 로컬 머신에서 실행되는 로컬 프로세스용

/mcp 명령으로 연결된 서버 확인, 상태 점검, 불필요한 서버 비활성화를 할 수 있습니다.

서버 범위 설정

로컬
현재 프로젝트에서만 사용 가능, 나만의 설정
사용자
모든 프로젝트에서 사용 가능
프로젝트
.mcp.json 파일을 버전 관리에 체크인해 코드베이스 모든 사람이 자동으로 동일한 서버를 사용

컨텍스트 비용

MCP 서버는 사용하지 않을 때도 도구 정의를 컨텍스트 윈도우에 추가합니다. 서버가 많으면 사용 가능한 컨텍스트를 소비합니다. /mcp로 연결된 서버를 확인하고 현재 사용하지 않는 것은 비활성화하세요.

⚠️ 도구 검색 모드 주의
MCP 도구가 컨텍스트 윈도우의 10%를 초과하면 Claude Code가 자동으로 도구 검색 모드로 전환합니다. 이는 필요한 도구를 수요에 따라 발견하지만 신뢰성이 다소 낮을 수 있습니다.
RECAP
  • claude mcp add로 MCP 서버를 추가합니다
  • .mcp.json으로 프로젝트 범위로 설정해 팀과 공유하세요
  • 현재 사용하지 않는 서버를 비활성화해 컨텍스트 사용량을 관리하세요
LESSON 12

Hooks

원본: https://anthropic.skilljar.com/claude-code-101/469798
WHAT YOU'LL LEARN
  • 훅이 무엇이고 왜 사용하는지
  • 사용 가능한 이벤트 유형
  • PreToolUse 훅으로 위험한 동작 차단
  • 팀과 훅 공유하는 방법

훅(Hooks)이란?

훅은 Claude Code의 라이프사이클의 특정 지점에서 명령을 실행할 수 있게 합니다. 훅과 이 강의에서 다룬 다른 모든 것의 핵심 차이점은 훅이 결정론적이라는 점입니다 — 항상 실행됩니다.

왜 훅을 사용하는가?

CLAUDE.md에 모든 파일 편집 후 Prettier를 실행하도록 지시할 수 있습니다. 대부분의 경우 실행합니다. 하지만 가끔은 그렇지 않습니다. 훅은 예외 없이 항상 실행되도록 합니다.

일반적인 사용 사례:

  • ▸ 파일 편집 후 자동 포맷팅
  • ▸ 컴플라이언스를 위한 실행된 명령 로깅
  • ▸ 프로덕션 파일 수정과 같은 위험한 동작 차단
  • ▸ 작업 완료 시 알림 전송

사용 가능한 이벤트

훅은 settings.json에 설정합니다. 이벤트를 선택하고, 적용할 도구를 위한 매처를 설정하고, 실행할 명령을 제공합니다:

이벤트 설명
PreToolUse 도구 호출 전 실행
PostToolUse 도구 호출 완료 후 실행
UserPromptSubmit 프롬프트 제출 시 Claude 처리 전 실행
Stop Claude가 응답 완료 시 실행
Notification Claude가 알림 전송 시 실행

실용적인 예시: 자동 포맷팅

가장 일반적인 훅: 편집 후 자동 포맷팅. "Edit|MultiEdit|Write" 매처가 있는 PostToolUse 훅을 설정해 Claude가 파일을 수정할 때마다 실행되도록 합니다. 명령은 파일 확장자를 확인하고 적절한 포맷터를 실행합니다.

PreToolUse로 차단하기

PreToolUse 훅은 도구 호출이 실행되기 전에 차단할 수 있습니다:
