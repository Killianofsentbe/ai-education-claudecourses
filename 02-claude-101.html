레슨 1-2 · What is Claude

Claude란? — 생각하는 파트너와의 첫 대화

Claude가 단순한 챗봇을 넘어 복잡한 문제를 함께 해결하는 사고 파트너인 이유와, 효과적인 첫 대화를 위한 프롬프트 방법론을 배웁니다.

📺 레슨 1 영상 →📺 레슨 2 영상 →

🎯 학습 목표

  • Claude의 설계 원칙(도움이 되고, 무해하며, 정직한)을 설명할 수 있다
  • Claude의 핵심 역량과 접근 방법(웹, 데스크탑, 모바일)을 파악한다
  • 명확한 맥락 설정 + 작업 정의 + 규칙 명시로 효과적인 프롬프트를 작성한다
  • 파일 업로드와 후속 메시지로 대화를 반복하며 개선할 수 있다

💡 핵심 요약

  • Constitutional AI: Claude는 인간의 가치와 일치하도록 학습된 원칙 기반 AI — 해로운 출력 최소화
  • Claude는 단순 Q&A를 넘어 요약, 검색, 창작 글쓰기, 코딩, 복잡한 추론까지 가능한 사고 파트너
  • 효과적인 프롬프트의 3요소: 맥락 설정(Setting the stage) + 작업 정의(행동) + 규칙 명시(형식·톤)
  • 대화는 반복적 과정 — 첫 번째 응답이 완벽하지 않아도 정상. 대화로 개선
  • 200K+ 토큰 컨텍스트 창으로 500페이지 이상의 문서를 단일 대화에서 처리 가능

✍️

글쓰기 & 콘텐츠

소셜 미디어 포스트부터 전문 보고서까지. 톤과 스타일을 반복적으로 조정 가능

🔬

리서치 & 분석

문서 업로드 후 복잡한 정보 파악. 대형 컨텍스트 창으로 방대한 자료 처리

💻

코딩 지원

여러 언어로 코드 작성, 디버깅, 설명. Claude Opus 4.5는 실제 코딩 작업에서 세계 최고 수준

🧠

문제 해결

복잡한 인지 작업, 수학 문제, 전략적 분석. 확장 사고 모드로 단계별 심층 추론

📚

학습 지원

새로운 기술, 낯선 도메인 탐색. 학습 스타일과 페이스에 적응

효과적인 프롬프트 작성법

Claude와 대화할 때는 동료에게 이야기하듯 자연스럽고 간결하게. 다음 세 요소를 고려하세요:

요소의미예시 질문
맥락 설정 (Setting the stage) 역할, 목표, 맥락 제공 "나는 인디 스트리밍 스타트업의 마케팅 리드이며, 투자자 피치 덱을 준비 중입니다"
작업 정의 (Defining the task) 원하는 구체적 행동 "독립 영화 스트리밍 시장의 주요 트렌드, 경쟁사 포지셔닝, 성장 기회를 리서치해 주세요"
규칙 명시 (Specifying rules) 형식, 톤, 예시 지정 "현재 웹 리서치와 인용 포함, 전문 보고서 형식으로 구성해 주세요"
완성된 예시 프롬프트
"저는 인디 스트리밍 스타트업의 마케팅 리드로, 투자자 피치 덱을 준비하고 있습니다. 현재 독립 영화 스트리밍 시장 조사를 통해 주요 트렌드, 경쟁사 포지셔닝, 성장 기회를 파악해 주세요. 현재 웹 리서치와 인용 출처를 포함하고, 전문 보고서 형식으로 구성해 주세요."

  • 코드 파일 → 작동 방식 설명 또는 버그 찾기

레슨 3 · Getting Better Results

더 나은 결과 얻기: 문제 해결과 AI Fluency

일반적인 Claude 사용 문제를 진단하고 해결하는 방법, 그리고 반복 마인드셋으로 지속적으로 더 나은 결과를 얻는 방법을 학습합니다.

📺 원본 영상 →

🎯 학습 목표

  • 흔한 AI 사용 문제를 인식하고 수정 기법을 적용할 수 있다
  • AI Fluency의 4D 프레임워크를 정의하고 자신의 활용에 연결한다
  • 특정 워크플로우에서 Claude 성능을 평가하는 간단한 방법을 갖춘다

💡 핵심 요약

  • 첫 번째 프롬프트가 완벽한 결과를 내지 못해도 정상 — 반복 마인드셋이 핵심
  • AI Fluency의 4D: Delegation(위임) + Description(설명) + Discernment(식별) + Diligence(성실)
  • 가장 중요한 교훈: 특정 고위험 정보는 항상 독립적으로 검증해야 함
  • 간단한 Eval 접근법으로 Claude가 내 업무에 얼마나 적합한지 체계적으로 파악 가능

흔한 문제와 해결책

문제원인해결책
너무 일반적인 응답 맥락이 부족한 프롬프트 대상, 역할, 제약 조건 추가. "이메일 써줘" → "엔터프라이즈 고객에게 2주 지연을 설명하는 전문적이지만 사과의 어조의 이메일 써줘"
너무 길거나 짧은 응답 적절한 길이를 추측함 명시적으로: "두 단락 요약" / "100 단어 이하" / "포괄적 분석 — 길이 제한 없음"
형식이 맞지 않음 무엇을 원하는지 알지만 어떻게 원하는지 모름 예시 제공 또는 구체적 설명: "각 섹션에 굵은 제목과 함께 글머리 기호 사용"
자신감 있어 보이는 오류 정보 특정 사실이나 틈새 주제에서 오류 발생 가능 고위험 업무 시 핵심 사실 독립 검증. 출처 인용 또는 신뢰도 표시 요청
톤이 맞지 않음 기본값이 도움이 되는 전문적 톤 "더 대화체로" / "권위 있고 공식적으로". 원하는 스타일의 글 예시 제공

AI Fluency — 4D 프레임워크

Prof. Rick Dakan과 Prof. Joseph Feller가 공동 개발한 프레임워크:

🎯

Delegation (위임)

어떤 업무를 AI에게, 어떤 것은 사람이 할지 결정. AI 역량 이해와 전략적 분배

💬

Description (설명)

AI와 효과적으로 소통. 결과물 정의, 프로세스 안내, 행동 방식 지정

🔍

Discernment (식별)

AI 결과물을 비판적 평가. 품질, 정확성, 적절성 판단 및 개선 영역 파악

🛡️

Diligence (철저함)

AI를 책임감 있고 윤리적으로 사용. AI 역할 투명하게 공개, AI 지원 업무 책임

내 워크플로우에서 Claude 평가하기 (Evals)

Claude가 특정 반복 업무에 얼마나 효과적인지 체계적으로 평가하는 방법:

  • 1

    예시 수집

    내가 정기적으로 하는 업무의 예시 5~10개 수집 (내가 작성한 이메일, 보고서, 분석 등)

  • 2

    테스트 프롬프트 작성

    유사한 결과물을 생성할 프롬프트 작성. 업무 시 자연스럽게 갖게 되는 맥락 포함

  • 3

    결과 비교

    Claude 응답을 내 예시와 비교: 핵심 정보를 담았는가? 톤과 스타일이 적절한가? 무엇이 빠졌는가?

  • 4

    접근법 개선

    배운 것을 바탕으로 프롬프트 수정, 더 많은 예시 추가, 인간 검토가 필수적인 부분 파악

🤔 성찰 질문

  • 이미 경험한 흔한 문제 중 어떤 수정 기법을 다음에 시도해볼 것인가요?
  • 4D 프레임워크 중 가장 자연스럽게 실천하는 것은? 가장 발전이 필요한 것은?
  • 반복 업무 중 간단한 Eval로 Claude 적합성을 테스트해볼 만한 것은 무엇인가요?
레슨 4 · Desktop App

Claude 데스크탑 앱: Chat·Cowork·Code

세 가지 모드의 역할과 고유 기능을 이해하고, 작업 유형에 맞는 모드를 선택하는 방법을 배웁니다.

📺 원본 영상 →

🎯 학습 목표

  • Chat, Cowork, Code 세 모드와 각 모드의 설계 목적을 설명할 수 있다
  • 각 모드의 고유 기능(빠른 입력, 예약 작업, 로컬/원격 개발 등)을 파악한다
  • 특정 업무에 맞는 모드를 선택할 수 있다

💡 핵심 요약

  • Chat: claude.ai와 동일한 Claude + 빠른 입력, 스크린샷, 받아쓰기, 데스크탑 커넥터
  • Cowork: 복잡한 리서치·분석·문서 작성에 특화. 폴더 접근, 예약 작업, 브라우저 사용 지원
  • Code: 코딩 전용 환경. Claude Code 기반, 시각적 diff, git 통합, 로컬/원격 환경
  • Cowork와 Code는 같은 엔진(Claude Code) 사용 — 둘 다 독립적 장기 작업 처리 가능
  • Cowork는 현재 연구 미리보기 (Pro, Max, Team, Enterprise 사용자)

세 모드 비교

항목 Chat Cowork Code
최적화 대상 빠른 교환: 아이디어 탐색, 초안 작성, 빠른 답변 복잡하거나 지속적인 작업: 리서치, 분석, 완성된 문서 소프트웨어 개발: 코드 작성, 테스트, 배포
주요 기능 빠른 입력, 받아쓰기, 스크린샷 폴더 접근, 플러그인, 서브에이전트, 예약 작업 Ask/Code/Plan 모드, 시각적 diff, git 통합
파일 시스템 커넥터로 클라우드 접근 공유 폴더 내 보호 환경 로컬 프로젝트 전체 파일 시스템 접근

레슨 5 · Projects

프로젝트: 지속적인 작업을 위한 전용 공간

관련 대화, 지식 베이스, 맞춤 지시사항을 하나의 프로젝트로 묶어 Claude를 더 똑똑하게 활용하는 방법을 배웁니다.

💡 핵심 요약

  • 프로젝트 = 자체 메모리, 대화 기록, 지식 베이스, 맞춤 지시사항을 갖춘 독립 작업 공간
  • 같은 파일을 반복 업로드하거나 매번 동일한 맥락을 설명해야 하는 경우 프로젝트가 해결책

  • 팀 사용자(Claude for Work): 프로젝트를 팀원과 공유해 동일한 맥락, 지시사항, 지식 활용
  • 파일 이름을 설명적으로 지정: Claude가 파일 이름으로 관련 정보를 찾음

프로젝트 생성 단계

  • 1

    프로젝트 설정

    왼쪽 사이드바에서 "Projects" 클릭 → "+ New Project" → 설명적인 이름 입력 → 가시성 설정 (비공개 또는 조직 공유)

  • 2

    프로젝트 지시사항 작성

    "Instructions" 클릭. 작업 맥락, 프로세스 지시사항, 톤/스타일, 특정 요건을 포함하세요. "Save instructions" 클릭 — 이 프로젝트의 모든 대화에 적용됩니다.

  • 3

    지식 베이스 구축

    오른쪽의 파일 메뉴에서 "+" 클릭. PDF, DOCX, CSV, TXT, HTML 등 업로드. Google Drive 연결 가능. 파일 이름은 설명적으로 지정 ("Q4-2024-브랜드가이드.pdf"가 "document1.pdf"보다 효과적)

프로젝트 활용 사례

프로젝트 이름지식 베이스에 포함할 내용지시사항 예시
Q4 제품 출시 제품 사양, 경쟁사 분석, 메시징 브레인스토밍 노트 "항상 B2B 기술 제품에 적합한 전문적이되 친근한 톤 사용"
고객 계정 허브 브랜드 가이드라인, 과거 결과물, 커뮤니케이션 이력 "제안서나 보고서 작성 시 항상 클라이언트 톤을 반영하고 특정 맥락 참조"
리서치 지원 경쟁사 검토, 사용자 리서치 데이터, 고객 피드백 "출처 종합, 보고서 초안, 추천 사항 일관성 유지"

프로젝트 vs 일반 대화

일회성 질문에는 일반 대화로, 지속적인 작업 스트림에는 프로젝트로. 같은 파일을 3번 이상 업로드했다면 프로젝트를 만들 시간입니다.

아티팩트와 스킬: 독립 결과물과 전문화 역량

📺 레슨 6 영상 →📺 레슨 7 영상 →

🎯 학습 목표

  • 아티팩트가 무엇인지, 언제 Claude가 아티팩트를 생성하는지 설명할 수 있다
  • 동료와 아티팩트를 공유하고 공개적으로 게시하는 방법을 파악한다
  • 스킬과 프로젝트의 차이를 이해하고, 맞춤 스킬을 만드는 방법을 안다

💡 핵심 요약

  • 아티팩트 = 대화 옆 전용 창에 렌더링되는 독립 실행 가능 결과물 (코드, HTML, 문서, 다이어그램 등)
  • 15줄 이상, 자체 완결적, 반복 사용 가능성이 있을 때 자동으로 아티팩트 생성
  • 공개 게시 시: 해당 버전만 공개, 대화는 비공개 유지. 다른 사람이 "리믹스" 가능
  • 스킬: 특정 작업을 반복 가능한 방식으로 실행하는 지시사항·스크립트·리소스 패키지
  • 프로젝트 vs 스킬: 프로젝트는 지식 저장, 스킬은 프로세스 실행

Claude를 일상적으로 사용하는 도구와 연결하여 실제 데이터로 작업하는 방법과, 조직 전체의 지식을 검색하는 엔터프라이즈 기능을 학습합니다.

🎯 학습 목표

엔터프라이즈 검색 활용 사례

엔터프라이즈 검색은 여러 출처에 걸친 질문이나 정보 종합이 필요할 때 특히 유용합니다:

사용 상황예시 질문

정책 & 프로세스"우리 회사 원격 근무 정책은?" / "경비 보고서는 어떻게 제출하나요?"

보안 및 권한

엔터프라이즈 검색은 내가 원래 연결된 도구에서 이미 접근 권한이 있는 내용만 표시합니다. 대화는 비공개이며 연결된 데이터는 별도로 인덱싱되거나 저장되지 않습니다.

리서치 모드: 심층 조사의 새로운 기준

단순 검색을 넘어 체계적인 다단계 조사를 자동으로 수행하는 리서치 모드의 활용법과, 직무별 실전 사례를 알아봅니다.

📺 레슨 10 영상 →📺 레슨 11 영상 →

🎯 학습 목표

  • 리서치 모드가 일반 웹 검색 및 확장 사고와 어떻게 다른지 설명할 수 있다
  • 종합적인 정보 수집이 필요한 상황에서 리서치 모드를 효과적으로 활용한다
  • 자신의 직무에 맞는 Claude 활용 사례 2~3가지를 파악한다

💡 핵심 요약

  • 리서치 모드 = Claude가 에이전트처럼 작동 — 여러 검색을 연쇄적으로 수행하며 스스로 무엇을 다음에 조사할지 결정
  • 대부분 5~15분 내 완료, 복잡한 조사는 최대 45분 — 수동 리서치라면 몇 시간이 걸릴 작업
  • 확장 사고가 자동으로 활성화 — Claude가 방법을 먼저 계획하고 정보를 수집
  • 모든 주장에 인용 출처 포함 — 검증 용이
  • 구글 워크스페이스 같은 통합 연결 시 내부 자료 + 웹 리서치 결합 가능

언제 무엇을 쓸까?

상황권장 도구이유
여러 출처의 종합 보고서 리서치 모드 자동으로 여러 검색 연쇄 실행, 인용 포함
오늘 주가 같은 빠른 사실 웹 검색 한두 개 출처로 충분, 속도가 중요
수학 문제, 코드 디버깅, 논리 분석 확장 사고 외부 정보보다 추론이 필요한 경우
