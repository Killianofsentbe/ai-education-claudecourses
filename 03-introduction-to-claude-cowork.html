레슨 1 · What is Cowork?

Cowork란?

예상 소요시간: 15분

📺 원본 영상 →

🎯 학습 목표

  • Cowork가 Claude 채팅과 어떻게 다른지 설명할 수 있다
  • 지식 업무에서 Cowork가 유용한 핵심 기능을 설명할 수 있다
  • Cowork가 적합한 작업과 일반 채팅이 적합한 작업을 구분할 수 있다

💡 핵심 요약

  • → Cowork는 Claude Code와 동일한 아키텍처 기반 — 지식 업무(분석·리서치·문서 작업)에 적용
  • → 대화에서 위임으로의 전환 — 텍스트 교환이 아닌 '완성된 파일'을 드라이브에 저장
  • → Cowork는 작업이 있는 곳에서 작동 — 파일 읽기/쓰기, 도구 연결, 장시간 작업 지속
  • → 사용자가 통제권 유지 — 작업 시작 전 계획 확인, 언제든 개입·방향 전환 가능

채팅과 Cowork의 차이

Claude 채팅 경험은 이미 익숙할 것입니다: 질문하면 답변이 오고, 문서를 붙여 넣으면 요약이 나옵니다. 하지만 도구 간 정보 이동, 출력 조합, 중간 단계 처리는 여전히 사용자 몫입니다.

Cowork는 이 교환 방식을 바꿉니다. 결과를 설명하면 Claude가 단계를 계획하고, 실행하고, 드라이브에 파일을 저장합니다.

  • 1

    Plan (계획)

    다중 단계 작업의 경우 Claude가 시작 전 접근 방식을 보여줍니다. 검토 후 필요 시 수정, 승인.

  • 2

    Execute (실행)

    필요한 시간 동안 격리된 환경에서 실행. 파일 생성·분석·장기 작업 — 다른 일 하면서 기다릴 수 있습니다.

  • 3

    Connect (연결)

    이메일, 공유 드라이브, 이미 로그인한 도구에 접근. 수동 복붙 없이 컨텍스트 자동 전달.

Cowork가 할 수 있는 것

🔗

커넥터

이메일, 메시지, 공유 드라이브 등 기존 도구에 연결. 컨텍스트가 자동으로 흘러들어 옵니다.

📁

파일 작업

드라이브의 프레젠테이션·스프레드시트·문서를 읽고, 편집하고, 생성. 완성된 파일이 드라이브에 저장됩니다.

🔌

플러그인

도메인 전문 지식을 내장한 Cowork. 역할·도메인에 맞는 워크플로우로 전문가처럼 접근합니다.

⏰

예약 작업

반복 주기로 워크플로우를 자동 실행. 매번 프롬프트 없이 정기 작업 수행.

🤖

서브에이전트

독립적인 작업들을 병렬로 처리. 여러 항목이 있을 때 동시에 작업합니다.

💻

로컬 컴퓨팅

파일에서 직접 코드 실행 후 결과를 다시 저장. 업로드·다운로드 없이 제자리에서 처리.

Cowork를 선택해야 할 때

유용한 질문: 이 작업이 내 파일, 연결된 도구, 또는 실제 출력 파일이 필요한가? 그렇다면 Cowork가 적합합니다.

Cowork 선택채팅 선택
완성된 파일(열 수 있는)이 필요할 때답변이나 직접 다듬을 초안이 필요할 때
드라이브 파일이나 연결 도구를 건드릴 때단일 붙여넣기로 필요한 모든 게 해결될 때
단계가 많거나 처리할 항목이 많을 때턴 바이 턴으로 함께 고민하고 싶을 때
실행 중 다른 일을 하고 싶을 때채팅 창에 한 번 붙여넣기로 충분할 때
레슨 2 · Getting set up

시작 설정

예상 소요시간: 7분

📺 원본 영상 →

🎯 학습 목표

  • Cowork에 접속하고 작업 폴더를 지정할 수 있다
  • 업무가 있는 도구와 페이지를 연결할 수 있다
  • 필요한 사전 조건을 이해한다

💡 핵심 요약

  • → Cowork는 Claude Desktop 앱에서 실행 — 유료 플랜 필요
  • → 폴더 지정이 채팅과의 핵심 차이 — 업로드 없이 직접 파일 읽기/쓰기
  • → 커넥터로 Slack, Google Drive, Gmail 등 한 번 설정하면 매 작업마다 활용 가능
  • → Claude in Chrome 확장으로 커넥터가 없는 내부 대시보드·벤더 포털도 접근 가능

준비 사항

Cowork는 Claude Desktop 앱에서 실행됩니다. 유료 Claude 플랜이 필요합니다. 현재 플랜 상세 정보를 확인하세요.

💡

Cowork가 보이지 않는다면

유료 플랜인지, 데스크탑 앱이 최신 버전인지 확인하세요.

Cowork 열기

Claude Desktop을 열면 상단에 Chat / Cowork / Code 모드 선택기가 있습니다. Cowork를 클릭하세요.

폴더 지정

채팅과의 핵심 차이: Claude에게 작업 디렉터리를 지정하면 해당 폴더에서 파일을 직접 읽고, 생성하고, 편집할 수 있습니다. 업로드도, 다운로드도 없습니다.

폴더에서 작업을 클릭하고 컴퓨터의 디렉터리를 선택하세요. Claude는 내부의 모든 파일(PDF, 스프레드시트, Word 문서 등)을 읽고, 완성된 작업을 같은 위치에 저장합니다. 변경 전에는 항상 허락을 구합니다.

도구 연결 & Chrome 확장

커넥터는 Cowork를 Slack, Google Drive, Gmail, Calendar 등 업무 서비스에 연결합니다. 한 번 설정하면 이후 모든 작업에서 자연스럽게 참조할 수 있습니다.

Cowork의 Customize 영역에서 목록을 확인하고 사용하는 도구를 켜세요.

커넥터가 없는 내부 대시보드·벤더 포털은 Claude in Chrome 확장으로 접근합니다. 같은 Customize 영역에서 설치할 수 있습니다.

레슨 3 · The task loop

태스크 루프

예상 소요시간: 15분

📺 원본 영상 →

🎯 학습 목표

  • 유용한 계획을 얻을 수 있을 만큼 명확하게 작업을 설명할 수 있다
  • 작업 시작 전·진행 중 방향을 조정할 수 있다
  • 완성된 출력물을 올바른 검토 마인드셋으로 접근할 수 있다

💡 핵심 요약

  • → Claude는 행동 전 계획을 제안하고 승인을 기다림 — 모든 Cowork 작업의 기본 패턴
  • → 좋은 프롬프트: 무엇을 볼 것인지 + 무엇을 돌려받을 것인지 + 어디에 저장할 것인지
  • → 진행 패널로 각 단계 모니터링 가능 — 실행 중에도 채팅으로 방향 수정 가능
  • → 결과물은 드래프트로 접근 — 능숙한 동료의 첫 번째 초안으로 읽고 다듬기

핵심 루프

  • 1

    원하는 결과물 설명

    무엇을 볼 것인지, 무엇을 돌려받을 것인지, 어디에 저장할 것인지를 담으세요. Claude가 빠진 부분은 후속 질문으로 채웁니다.

  • 2

    몇 가지 질문에 답하기

    프롬프트와 파일을 바탕으로 Claude가 출력물 방향을 잡기 위한 질문을 합니다. 제시된 옵션 중 선택하거나 직접 답변하세요.

  • 3

    자리 비우기 — 또는 개입하기

    진행 패널이 각 단계를 보여줍니다. 자리를 비워도 되고, 채팅에 입력해 방향을 바꿀 수도 있습니다.

  • 4

    완성된 작업물 열기

    결과물이 지정된 폴더에 저장됩니다. 능숙한 동료의 첫 번째 초안처럼 — 좋은 작업물이지만 여전히 다듬을 여지가 있습니다.

실전 예시: 산만한 폴더 → 완성된 프레젠테이션

회의 메모, 체크리스트, 타임라인 스프레드시트, 저장된 이메일, 비교 매트릭스가 섞인 프로젝트 폴더가 있다고 가정합니다.

프롬프트 예시
"이 폴더의 모든 내용을 검토하고 툴링 리뷰를 위한 리더십 프레젠테이션을 만들어 주세요. 벤더 평가 결과, 타임라인, 비즈니스 케이스, 미해결 리스크를 포함하고 PowerPoint 파일로 출력해 주세요."

Cowork가 계획을 보여주면 검토 후 승인. 작업이 완료되면 실제 PowerPoint 파일이 폴더에 저장됩니다. 차트는 편집 가능한 요소로 클릭해서 조정할 수 있습니다.

서브에이전트와 병렬 처리

독립적인 부분이 있는 작업은 Cowork가 별도 워크스트림을 동시에 실행할 수 있습니다. 예: 4개 벤더 비교 시 벤더별 서브에이전트가 각각 가격·통합·리뷰를 리서치하고 결과를 하나로 합칩니다.

레슨 4 · Giving Cowork context

컨텍스트 제공

예상 소요시간: 10분

📺 원본 영상 →

🎯 학습 목표

  • 매 세션 재설명 없이 지속되는 컨텍스트를 Cowork에 제공할 수 있다
  • 프로젝트의 Instructions 패널을 활용해 작업 방식을 Claude에게 알릴 수 있다
  • 모든 곳에 적용되는 글로벌 지침을 설정할 수 있다

💡 핵심 요약

  • → 각 Cowork 태스크는 새로 시작 — 이전 대화 메모리 없음. 컨텍스트 유지 수단: 프로젝트
  • → 프로젝트 = 로컬 폴더 기반 작업 공간 + 모든 태스크에 적용되는 지침·메모리
  • → Instructions 패널에 담을 것: 관련 인물, 파일 위치, 출력 형식, 프로젝트 규칙
  • → 글로벌 지침(Settings → Cowork)은 프로젝트 불문 모든 태스크에 적용

왜 중요한가

각 Cowork 태스크는 새로 시작합니다. Claude는 이전 태스크의 대화 메모리를 가져오지 않습니다. 컨텍스트가 유지되는 방법은 프로젝트: 실제 로컬 폴더를 기반으로 하며, 지침과 메모리가 그 안에서 시작하는 모든 태스크로 이어집니다.

프로젝트는 Cowork 사이드바에 있습니다. 처음부터 만들거나, Chat 프로젝트에서 가져오거나, 기존 폴더를 래핑할 수 있습니다.

Instructions에 담을 내용

프로젝트 안에 들어가면 오른쪽 패널에 Instructions 섹션이 있습니다. 여기에 쓴 내용이 그 프로젝트에서 시작하는 모든 태스크에서 읽힙니다.

  • 관련 인물 — 이름, 역할, 연락 방법. "이걸 Rachel에게 보내줘"가 의미 있어집니다
  • 파일 위치 — "계약서는 ./Contracts, 오래된 보고서는 Drive /archive/[year]에"
  • 출력 형식 — "초안은 .docx, 최종본은 PDF, deliverables 서브폴더에 저장"
  • 프로젝트 규칙 — "미터 단위 사용, 모든 숫자에 출처 인용"
💡

확인 방법

프로젝트 안에서 물어보세요: "내가 여기서 어떻게 일하는지 알고 있는 것을 말해줘." Instructions와 메모리가 잘 반영되고 있는지, 무엇이 빠졌는지 확인할 수 있습니다.

글로벌 지침

프로젝트 간에 변하지 않는 기본값 — 역할, 기본 출력 형식, "삭제 전 물어봐" 같은 규칙 — 은 Settings → Cowork → Global Instructions에서 설정하세요. 프로젝트 안팎 모든 Cowork 태스크에 적용됩니다.

레슨 5 · Plugins: Cowork as a specialist

플러그인: 전문가 Cowork

예상 소요시간: 15분

📺 원본 영상 →

🎯 학습 목표

  • 플러그인이 무엇이고 내부 구성이 어떻게 되는지 설명할 수 있다
  • 자신의 역할에 맞는 플러그인을 설치할 수 있다
  • 스킬이 플러그인의 핵심 구성 요소임을 이해한다

💡 핵심 요약

  • → 플러그인 = 역할/도메인을 위한 스킬 + 커넥터 + 서브에이전트 번들
  • → 영업·마케팅·제품·재무·법무 등 대부분 지식 업무 역할에 오픈소스 플러그인 제공
  • → 플러그인은 컴퓨터의 폴더 — 모든 파일이 일반 텍스트라 직접 읽고 편집 가능
  • → 스킬은 Claude가 특정 작업을 처리하는 방법을 담은 마크다운 파일 — Cowork·Chat·Code 등 모든 Claude 인터페이스에서 작동

플러그인이란

플러그인은 Cowork에 도메인 전문 지식을 부여합니다. 각 플러그인은 특정 기능에 맞는 지식과 워크플로우를 내장해, Claude가 해당 역할의 전문가처럼 작업에 접근하게 합니다.

📋

스킬

특정 워크플로우 처리 지침. Claude가 자동으로 활용하거나 /로 직접 호출. 예: /prep-call, /weekly-report

🔗

커넥터

작업이 일어나는 시스템에 연결. 예: CRM, 문서, 메시지

🤖

서브에이전트

전문화된 작업을 병렬 처리. 예: 전체 계정 리뷰 시 계정별 에이전트

플러그인 내부 구조

플러그인은 단순히 폴더입니다:

sales-plugin/
├── plugin.json      ← 매니페스트: 이름, 설명, 의존성
├── skills/
│   ├── deal-brief/   ← 딜 브리프 구성법
│   └── prep-call/   ← 슬래시 메뉴의 /prep-call
└── agents/
    └── account-sweep.md ← 계정별 서브에이전트

모든 파일은 일반 텍스트입니다. 스킬 동작 방식을 바꾸려면 파일을 열고 편집하세요. 빌드 단계 없이 Cowork가 폴더를 직접 읽습니다.

설치 방법

  • 1

    탐색

    Cowork Customize 영역에서 플러그인 목록 확인

레슨 6 · Scheduled tasks

예약 작업

예상 소요시간: 8분

📺 원본 영상 →

🎯 학습 목표

  • 반복 주기로 자동 실행되는 예약 작업을 설정할 수 있다
  • 앱이 닫혀 있거나 컴퓨터가 절전 상태일 때 예약 작업의 동작을 이해한다

💡 핵심 요약

  • → 예약 작업 = 잘 만들어진 Cowork 태스크를 시간표에 따라 자동 실행
  • → 설정: /schedule 입력 또는 사이드바의 예약 작업 영역 이용
  • → 앱이 닫혀 있을 때는 지연 — 재오픈 시 즉시 실행 후 알림
  • → 스킬 + 예약 작업의 조합: 스킬이 무엇을 할지 정의, 예약이 언제 할지 결정

예약 작업이란

예약 작업은 모든 Cowork 태스크를 시간표에 따라 자동으로 실행합니다 — 시간별, 일별, 주별, 또는 커스텀 주기. 잘 작동하는 워크플로우가 생기면 매번 직접 프롬프트할 필요가 없어집니다.

"매주 월요일 오전 9시에 캘린더를 확인하고 주간 우선순위 이메일 초안을 작성해 줘."

설정 방법

Cowork 대화 창에 /schedule을 입력하거나, 사이드바의 예약 작업 영역을 이용하세요. Claude가 주기, 폴더, 출력 형태를 안내합니다.

반복 실행이므로 승인 단계가 있습니다. 승인 후에는 데스크탑 앱이 열려 있는 동안 자동으로 실행됩니다. 앱이 닫혀 있었다면 재오픈 시 즉시 실행하고 지연 사실을 알려줍니다.

스킬 + 예약 작업 조합

스킬은 무엇을 할지 정의하고, 예약 작업은 언제 할지 결정합니다. 평일 오전 8시로 예약된 브리핑 스킬은 매일 아침 브리핑이 기다리고 있음을 의미합니다.

💡

실천 팁

지금 당장 예약하지 않아도 됩니다. 반복하는 작업을 메모해 두고, Cowork에서 한 번 성공적으로 실행한 후 예약하는 것이 자연스러운 흐름입니다.

레슨 7 · File & document tasks

파일 & 문서 작업

예상 소요시간: 10분

📺 원본 영상 →

🎯 학습 목표

  • Cowork에 적합한 파일·문서 작업을 파악할 수 있다
  • 입력·변환·출력을 명시하는 프롬프트를 작성할 수 있다
  • 예시 프롬프트를 자신의 작업에 맞게 응용할 수 있다

💡 핵심 요약

  • → Cowork는 실제 파일을 생성 — 편집 가능한 차트가 있는 프레젠테이션, 수식이 있는 스프레드시트
  • → 출력은 네이티브 형식 — PowerPoint의 차트는 클릭해서 데이터·서식 직접 수정 가능
  • → 좋은 프롬프트 패턴: 입력(어디에서) + 변환(무엇을) + 출력(어떤 형식으로)

프롬프트 패턴

요소의미예시
입력소스 자료 위치"내 Downloads 폴더", "이 회의 메모", "이 폴더"
변환무엇을 할지"유형별 정렬", "슬라이드 덱으로 변환", "하나로 합치기"
출력어떤 형식으로"날짜별 서브폴더", "요약 뷰가 있는 스프레드시트", "형식화된 보고서"
⚠️

나쁜 예시 vs 좋은 예시

"내 파일 정리해줘" — 세 요소 없음, 명확화 질문 받음
"Downloads를 파일 유형별로 날짜 우선 서브폴더로 정렬해줘" — 세 요소 모두 있음, 바로 작업 시작

초반에 자주 쓰는 프롬프트

정리하기

"Downloads 폴더를 파일 유형별로 날짜 우선 서브폴더로 정렬해줘."
"이 폴더의 모든 파일을 일관된 날짜 우선 형식으로 이름 바꿔줘."
"이 폴더의 영수증에서 형식화된 지출 보고서를 만들어줘."

만들기

"이 메모에서 상태 컬럼과 요약 뷰가 있는 프로젝트 추적 스프레드시트를 만들어줘."
"이 회의 메모를 슬라이드 덱으로 만들어줘. brand-guidelines 스킬 사용해줘."
"이 폴더의 트랜스크립트와 메모를 형식화된 보고서로 합쳐줘."
레슨 8 · Research & analysis at scale

리서치 & 규모의 분석

예상 소요시간: 10분

📺 원본 영상 →

🎯 학습 목표

  • Cowork의 접근 방식이 가장 도움이 되는 리서치·분석 작업을 파악할 수 있다
  • 규모에서 Cowork가 표면화할 수 있는 인사이트 유형을 설명할 수 있다
  • 요약이 아닌 신호를 찾는 프레임으로 작업을 설계할 수 있다

💡 핵심 요약

  • → Cowork가 적합한 리서치: 볼륨(붙여넣기에 너무 많음), 병렬성(N개 항목 동시 분석), 제자리 컴퓨팅(업로드 없이 파일에서 직접 코드 실행)
  • → 규모의 가치: 모든 소스를 동시에 보유하므로 순차 읽기로는 놓치는 연결점 발견
  • → 가장 유용한 프롬프트: '요약해줘'가 아닌 내가 시간이 있다면 직접 답했을 질문

어떤 형태의 작업에 적합한가

📦

볼륨

붙여넣기에 너무 많거나 너무 큰 파일들. Cowork는 제자리에서 읽음 — 단일 채팅 한계 없음.

⚡

병렬성

"N개 항목에 동일 분석을 수행" — Cowork가 동시에 처리.

🖥️

제자리 컴퓨팅

파일이 있는 위치에서 코드 실행 후 결과를 디스크에 다시 저장. 업로드·다운로드 없음.

활용 예시

리서치 종합

"리서치 폴더의 모든 내용을 읽고 종합 보고서를 작성해줘. 출처들이 동의하는 부분은? 서로 모순되는 부분은? 단 하나의 출처만 주장하는 내용은 표시해줘."

인터뷰·회의 분석

"이 폴더의 모든 트랜스크립트를 읽어줘. 대부분의 사람들이 동의한 것은? 이견을 가진 사람들이 있다면 그들의 공통점은?"

데이터 분석

"이 파일의 각 고객에 대해 월별 사용량 변화를 계산해줘. 3개월 연속 감소한 계정을 표시하고 같은 폴더에 요약을 저장해줘."

신호를 찾는 프레이밍

가장 유용한 프롬프트는 "요약해줘"가 아닙니다. 시간이 있다면 직접 답했을 질문입니다:

막연한 질문날카로운 질문
"트랜스크립트 요약해줘""대부분이 동의한 것은? 이견을 가진 사람들의 공통점은?"
"데이터 분석해줘""지난 3개월 기준 위험 계정은? 공통 패턴은?"
"논문 검토해줘""이 논문들이 서로 모순되는 부분은? 가장 많은 단서가 필요한 주장은?"
레슨 9 · Permissions, usage, & choosing your model

권한·사용량·모델 선택

예상 소요시간: 10분

📺 원본 영상 →

🎯 학습 목표

  • Cowork가 운영되는 안전 경계를 설명할 수 있다
  • 사용량 할당을 효율적으로 관리할 수 있다
  • 출력물에 근거해 행동하기 전 검토하는 습관을 쌓을 수 있다

💡 핵심 요약

  • → 격리 실행 환경 + 허가된 폴더만 접근 + 영구 삭제는 항상 명시적 승인 필요
  • → Cowork는 채팅보다 할당량을 더 많이 사용 — 관련 작업 묶기, 채팅으로 대체 가능한 작업은 채팅 이용
  • → 모델 3단계: Opus(복잡한 다단계), Sonnet(일상 기본값), Haiku(빠르고 가벼운 작업)
  • → 출력물 검토 습관: 더 완성도 있어 보일수록 한 번 더 확인할 가치 있음

권한과 안전

🔒

격리 실행

컴퓨터의 격리 환경에서 실행. 허가된 것 외에는 접근 불가.

📂

제어된 파일 접근

어느 폴더를 볼 수 있는지 사용자가 결정. 허가 없으면 접근 없음.

🛡️

삭제 제어

영구 삭제는 명시적 승인 필요. 항상 먼저 확인 요청.

ℹ️

참고사항

Cowork 대화 기록은 컴퓨터 로컬에 저장됩니다. 규정 요구사항이 있는 경우 현재 플랜의 감사 로깅 및 컴플라이언스 기능을 확인하세요.

사용량 관리

  • 관련 작업 묶기 — 새 세션 시작에는 오버헤드가 있습니다. 관련 작업이 여러 개라면 한 세션에 처리하세요
  • 채팅으로 대체 — 파일, 연결 도구, 실제 출력 파일이 필요 없다면 채팅이 더 빠르고 덜 소모적입니다
  • 사용량 모니터링 — Cowork 설정에서 주기적으로 확인하세요

올바른 모델 선택

모델특징적합한 작업
Opus가장 강력, 가장 많은 할당량 사용복잡한 다단계 작업
Sonnet속도와 깊이의 균형 (기본값)일상적인 작업 대부분
Haiku가장 빠르고 가벼움간단하고 빠른 작업

모든 것에 가장 높은 옵션을 기본으로 사용하지 말고, 작업에 필요한 것에 맞는 모델을 선택하세요.

레슨 10 · Troubleshooting & next steps

문제 해결 & 다음 단계

예상 소요시간: 10분

📺 원본 영상 →

🎯 학습 목표

  • Cowork 사용 시 흔히 겪는 문제를 인식하고 해결할 수 있다
  • 설치 → 실행 → 예약 → 공유의 다음 단계 로드맵을 알고 있다
  • 이 강의 이후 도움받을 수 있는 곳을 안다

💡 핵심 요약

  • → 작업공간 준비 시간은 정상 — 특히 처음이나 업데이트 후 조금 더 걸릴 수 있음
  • → 작업 중간에 앱 종료 = 태스크 중단. 절전은 괜찮음 — 깨어나면 재개
  • → 파일을 못 찾는다면: Cowork가 사용 중이던 작업 폴더와 설정의 폴더 접근 권한 확인
  • → 목표: 예약된 스킬 하나 신뢰할 수 있게 실행 → 플러그인 설치 → 실제 작업 → 스킬 저장 → 예약 → 공유

흔한 문제와 해결

작업 공간 준비 중

시작 전 환경 준비 시간입니다. 첫 번째 실행이나 업데이트 후에는 조금 더 걸릴 수 있습니다. 기다리세요.

작업이 중간에 멈췄다

가장 흔한 원인: 작업 중 데스크탑 앱이 종료됨. 절전은 괜찮습니다 — 세션이 유지되고 깨어나면 재개됩니다. 앱이 열려 있다면 네트워크 연결을 확인하세요.

사용량 한도 초과

Cowork는 채팅보다 할당량을 더 소모합니다. 레슨 9의 팁: 관련 작업 묶기, 채팅으로 대체 가능한 작업 분리, 모델 선택 최적화.

Cowork가 만들었다는 파일을 찾을 수 없음

Cowork가 사용 중이던 작업 폴더를 확인하세요. 설정에서 폴더 접근 권한을 확인하세요. Cowork에게 직접 파일을 어디에 저장했는지 물어보세요.

다음에 할 일

  • 1

    플러그인 설치

    라이브러리를 탐색하고 내 역할에 맞는 플러그인을 설치. 플러그인 폴더를 열어 스킬 파일 하나를 읽어보세요.

  • 2

    실제 작업 실행

    실제 업무에서 의미 있는 작업을 선택해 실행. 자리를 비운 사이 완성된 파일이 기다리고 있는 순간을 경험하세요.

  • 3

    스킬 만들기

    반복하게 될 작업을 실행했다면 그 세션을 스킬로 저장하세요.

  • 4

    예약하기

    스킬을 시간표에 올리세요. 앱이 닫혀 있으면 재오픈 시 즉시 실행됩니다.

  • 5

    공유하기

    팀원에게 스킬을 패키지로 전달하거나, Team·Enterprise에서는 관리자에게 더 넓게 배포를 요청하세요.
